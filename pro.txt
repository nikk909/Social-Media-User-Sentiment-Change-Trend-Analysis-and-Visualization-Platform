社交媒体用户情绪变化趋势分析与可视化平台 - 开发需求文档
一、项目目标
开发一个轻量化平台，实现社交媒体用户情绪的动态追踪与可视化。核心价值是通过分析用户情绪随时间的变化趋势，为市场营销、政策制定、心理健康研究等场景提供数据支持，替代单一时间点的静态分析。
二、核心功能模块（按流程拆分，降低复杂度）
数据采集模块
功能：从主流社交媒体（如微博、Twitter）爬取指定话题 / 事件 / 群体的文本数据（含发布时间）。
要求：支持按时间范围筛选，数据格式包含「文本内容、发布时间、来源平台」。
技术选型：轻量级爬虫（如 Requests+BeautifulSoup，或 Scrapy 简化版），避免过度分布式设计。
文本预处理模块
功能：清洗文本（去除特殊符号、表情符号标准化、去重），保留时间戳用于后续时序分析。
输出：结构化数据（DataFrame 格式，含「cleaned_text、timestamp」字段）。
情感分类模块
功能：对清洗后的文本进行情感分类（正面 / 负面 / 中性）。
技术选型：优先使用预训练轻量模型（如 TextBlob 快速分词 + 情感打分，或简化版 VADER；若需更高精度，可集成轻量化 BERT 模型如 ALBERT），避免过度依赖 GPU（基础版本支持 CPU 运行）。
输出：新增「sentiment」字段（标记 pos/neg/neu）。
时间序列分析模块
功能：按时间窗口（如每小时 / 每天）聚合情感数据，计算各时段内正面 / 负面 / 中性情绪的占比；识别该时段内的热点话题（高频词）。
技术：用 Pandas 进行时间分组（resample）、统计计数；用 TF-IDF 提取各时段高频词。
输出：时序统计结果（含「时间窗口、pos_ratio、neg_ratio、neu_ratio、hot_words」）。
可视化模块
功能：通过交互式图表展示情绪趋势与热点话题。
核心图表：
折线图：展示正面 / 负面 / 中性情绪占比随时间的变化；
词云图：按情绪分类（或时间窗口）展示高频词；
简单热力图：展示不同话题的情绪强度分布（可选）。
技术选型：Plotly（支持交互）+ 简单 Web 界面（如 Streamlit，快速部署）。
三、数据流程（简化版）
原始文本数据 → 清洗预处理 → 情感分类 → 时序聚合统计 → 可视化展示
四、关键约束（降低实现难度）
优先支持单话题 / 事件分析，暂不考虑多话题并行；
时间窗口可固定（如按小时），避免动态调整复杂度；
初期不追求实时性，支持「批量导入历史数据 + 分析」即可，后续可扩展为定时爬取。
五、技术栈清单（轻量化优先）
数据采集：Requests + BeautifulSoup（或 Scrapy）
数据处理：Pandas（清洗、时序分析）
情感分析：TextBlob（入门）/ VADER（社交媒体适配）
文本特征：Scikit-learn（TF-IDF）
可视化：Plotly（绘图）+ Streamlit（简易界面）
